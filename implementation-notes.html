Implementation Notes

  • 本轮增强只处理仓库内可上线的静态站点内容；GitHub 域名验证、GitHub Pages Enforce HTTPS、Cloudflare SSL/TLS、缓存规则、安全响应头、Email Obfuscation 和 /cv 重定向都需要在对应后台手动配置。
  • Selected Works / Research Highlights 按用户要求暂不实现。
  • 邮箱防爬只按用户要求保留给 Cloudflare Email Obfuscation 处理，不再继续使用前端 JavaScript 拼接邮箱，避免 Cloudflare 无法识别邮箱文本。
  • 为了避免中国大陆访问受阻，本轮会移除 Google Fonts、Google Maps iframe 和 unpkg Ionicons CDN 依赖，站点改用系统字体和本地 SVG 图标。
  • CV 文件将采用版本化文件名，当前版本使用 KaiWu_CV_0705.pdf；旧文件是否保留取决于站点链接兼容性和仓库维护成本。
  • 新增的 Impressum 和 Privacy 页面是低成本公开说明，不构成法律意见；如需严格合规，应由熟悉德国 DDG/DSGVO 的专业人士审阅。
  • GitHub Actions 只做静态回归检查，不引入外链检查器，避免 CI 因外部网络波动误报。
  • 复查 Cloudflare 后发现：`about.wukai.work` 当前 CNAME 为 DNS-only，未走 Cloudflare 代理，因此 Email Obfuscation、Redirect Rules、Cache Rules、Response Header Transform Rules 等 Cloudflare 功能不会对该站点生效。
  • 当前 Cloudflare API token 可以读取 `wukai.work` DNS，但读取 SSL/TLS、Always Use HTTPS、Email Obfuscation、Brotli、Page Rules 和 Rulesets 时返回 403 权限错误；因此本轮不能直接通过 API 落地这些 Cloudflare 配置。
  • HSTS `includeSubDomains`、HSTS preload 和激进 Bot/WAF 挑战规则可能影响其他 `wukai.work` 子域名或学术搜索/预览爬虫，本轮不擅自启用，需在确认所有相关子域名和 crawler 影响后再做。
  • 在 Cloudflare Redirect Rule 暂不可通过 API 配置的情况下，新增静态 `/cv/` fallback 页面指向当前版本化 CV；后续 Cloudflare 规则可继续配置 `/cv` 到 PDF 作为更直接的边缘重定向。
